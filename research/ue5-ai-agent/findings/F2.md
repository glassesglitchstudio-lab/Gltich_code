# F2: UE5 AI Agent Integration Research

**Date:** 2026-07-08 | **Confidence key:** High/Medium/Low

---

### 1. Existing UE5 AI Plugins

**Claim:** Mature open-source LLM plugins exist with MCP support.
- **UnrealGenAISupport** (623 stars): Multi-provider (OpenAI, Claude, Gemini, DeepSeek, Grok). MCP server for Claude Desktop/Cursor. UE 5.4-5.7+.
  - URL: https://github.com/prajwalshettydev/UnrealGenAISupport
- **SpecialAgentPlugin** (46 stars): MCP server with 71+ tools, 14 service categories. Python execution with full UE5 API.
  - URL: https://github.com/ArtisanGameworks/SpecialAgentPlugin
- **unreal-ollama** (16 stars): Local LLM via Ollama, multimodal support, UE 5.1-5.6.
  - URL: https://github.com/MuddyTerrain/unreal-ollama
- **Confidence:** High

### 2. LLM HTTP Integration Pattern

**Claim:** Async HTTP via FHttpModule with delegate callbacks is standard.
- Create request struct (model, messages) -> SendChatRequest() -> callback with response/error.
- API keys via env vars (PS_OPENAIAPIKEY), runtime injection for packaged builds.
- Streaming via chunked responses supported.
- **Confidence:** High

### 3. External CLI/Subprocess from UE5

**Claim:** FPlatformProcess::ExecProcess() or Python bridge pattern.
- SpecialAgentPlugin runs Python via UE5's PythonScriptPlugin.
- Critical: use AsyncTask(ENamedThreads::AnyBackgroundThreadNormalTask) to avoid game thread blocking.
- **Confidence:** Medium

### 4. WebSocket/HTTP Server Patterns

**Claim:** MCP over HTTP/SSE (JSON-RPC 2.0) is the emerging standard.
- SpecialAgentPlugin: port 8767, native HTTP/SSE, no dependencies.
- UnrealGenAISupport: Python MCP server on port 9877.
- Epic developing official MCP for UE 5.8+.
- **Confidence:** High

### 5. Memory/State Management

**Claim:** Current plugins are stateless; game state injected via tool calls.
- Conversation state held in FGenChatMessage[] structs (in-memory, per-session).
- SpecialAgentPlugin's 71+ tools expose world state to LLM on demand.
- No persistent memory layer in open-source plugins.
- **Confidence:** High

---

## Summary

MCP via HTTP/SSE is the standard pattern. For a custom agent: use async HTTP delegates, implement MCP server, keep state in memory, inject game context via tools. Consider unreal-ollama for local inference.
