# F1: UE5 Plugin Development Architecture

## Findings

### Plugin File Structure
**Claim**: UE5 plugins consist of a `.uplugin` JSON descriptor, `Source/` directory with `*.Build.cs` build rules, and optional `Content/`, `Config/`, `Shaders/` folders. Each plugin has at least one module.
**URL**: https://dev.epicgames.com/documentation/en-us/unreal-engine/creating-ue5-plugins
**Date**: 2024-12-01
**Confidence**: High

### Module Types
**Claim**: UE5 plugins use module types: `Runtime` (gameplay), `Editor` (unreal editor only), `UncookedOnly`, `Developer`, `ThirdParty`, and `EditorNoCommandlet`. Editor modules are excluded from packaged builds. A plugin can contain multiple modules for separation of concerns.
**URL**: https://dev.epicgames.com/documentation/en-us/unreal-engine/build-system-unreal-engine
**Date**: 2024-12-01
**Confidence**: High

### .uplugin Descriptor
**Claim**: The `.uplugin` file is JSON with fields: `Modules` (array with Name, Type, LoadingPhase), `Plugins` (dependencies), `Category`, `CreatedBy`, `CanContainContent`, `MarketplaceURL`, `EngineVersion`. LoadingPhase controls init order (e.g., `PostEngineInit` for editor UI).
**URL**: https://dev.epicgames.com/documentation/en-us/unreal-engine/creating-ue5-plugins
**Date**: 2024-12-01
**Confidence**: High

### Build.cs Configuration
**Claim**: `*.Build.cs` files use `ModuleRules` to declare `PublicDependencyModuleNames`, `PrivateDependencyModuleNames`, `IncludePaths`, and `PreprocessorDefinitions`. Editor modules typically depend on `UnrealEd`, `Slate`, `SlateCore`, `EditorStyle`.
**URL**: https://dev.epicgames.com/documentation/en-us/unreal-engine/unreal-engine-build-system
**Date**: 2024-12-01
**Confidence**: High

### Editor Extension Points
**Claim**: UE5 provides `FExtender` for toolbar/menu hooks, `IEditorModeTools` for viewport modes, `FGlobalTabmanager` for registration of tabs/panels, and `UFactory` for asset import. Toolbar buttons are registered via `UToolMenus::Get()->ExtendMenu()` or `FLevelEditorModule::ToolBarExtender`. Menu entries use `FMenuBuilder`.
**URL**: https://dev.epicgames.com/documentation/en-us/unreal-engine/unreal-engine-editor-scripting-and-automation-guide
**Date**: 2024-12-01
**Confidence**: High

### Custom Editor Window (SDockTab)
**Claim**: Custom editor windows are created by registering an `FGlobalTabmanager::FTabSpawnerEntry` with `RegisterTabSpawner`, returning an `SDockTab` wrapping an `SCompoundWidget` or `SWindow`. Slate is the C++ UI framework. Widgets compose via `SNew()`, `SLATE_BEGIN_ARGS`, and `SAssignNew`. Layout uses `SVerticalBox` / `SHorizontalBox`.
**URL**: https://dev.epicgames.com/documentation/en-us/unreal-engine/slate-ui-framework-in-unreal-engine
**Date**: 2024-12-01
**Confidence**: High

### Blueprint Integration
**Claim**: Plugins expose functionality to Blueprints by subclassing `UBlueprintFunctionLibrary` (static functions), `UBlueprintAsyncActionBase` (async nodes), or `UActorComponent`. `UFUNCTION(BlueprintCallable, Category=...)` exposes C++ functions. `UCLASS(Blueprintable)` allows Blueprint subclassing. `UEdGraphNode` subclass for custom graph nodes.
**URL**: https://dev.epicgames.com/documentation/en-us/unreal-engine/blueprint-visual-scripting-in-unreal-engine
**Date**: 2024-12-01
**Confidence**: High

### C++ vs Blueprint Tradeoffs
**Claim**: C++ plugins offer full engine access, better performance, compile-time safety, and access to editor APIs. Blueprint-only plugins are faster to prototype, artist-friendly, but cannot create editor UI, have runtime overhead, and cannot access internal engine APIs. Hybrid approach (C++ core + Blueprint-exposed API) is the standard production pattern.
**URL**: https://dev.epicgames.com/documentation/en-us/unreal-engine/introduction-to-blueprints-visual-scripting-in-unreal-engine
**Date**: 2024-12-01
**Confidence**: High
