# F3: External CLI Integration in UE5 Plugins

## 1. FPlatformProcess API for Running External Processes

**Claim:** UE5 exposes `FPlatformProcess::CreateProc()` for launching external executables. Two overloads exist: one with stdout/stderr pipes (`PipeWriteChild`, `PipeReadChild`, `PipeStdErrChild` parameters) and one without. Returns `FProcHandle` for lifecycle management.
**URL:** https://dev.epicgames.com/documentation/unreal-engine/API/Runtime/Core/GenericPlatform/FGenericPlatformProcess?application_version=5.4
**Date:** 2025
**Confidence:** High

## 2. Capturing stdout/stderr from External Processes

**Claim:** `FPlatformProcess::CreatePipe()` creates anonymous pipes (`void*& ReadPipe, void*& WritePipe`). Pass pipe handles to `CreateProc()` as child-side pipes. Read output via `FPlatformProcess::ReadPipe(void* ReadPipe)` which returns `FString`, or `ReadPipeToArray()` for raw `TArray<uint8>`. Close pipes with `ClosePipe()`.
**URL:** https://dev.epicgames.com/documentation/unreal-engine/API/Runtime/Core/GenericPlatform/FGenericPlatformProcess?application_version=5.4
**Date:** 2025
**Confidence:** High

## 3. Named Pipes and IPC in UE5

**Claim:** UE5 provides `NewInterprocessSynchObject()` for named semaphores. For bidirectional IPC, the standard pattern is anonymous pipe pairs via `CreatePipe()`. Named pipes require platform-specific implementation (Windows `CreateNamedPipe`/`ConnectNamedPipe`). No built-in UE5 named pipe abstraction exists beyond anonymous pipes.
**URL:** https://dev.epicgames.com/documentation/unreal-engine/API/Runtime/Core/GenericPlatform/FGenericPlatformProcess?application_version=5.4
**Date:** 2025
**Confidence:** Medium

## 4. HTTP Server Approach

**Claim:** Run CLI tool as a local HTTP server; UE5 connects via `FHttpModule` (`IHttpRequest`). This decouples processes, avoids blocking, and works cross-platform. UE5's HTTP module supports async requests with callbacks.
**URL:** https://forums.unrealengine.com/t/work-with-the-http-protocol-in-ue4/290759/7
**Date:** 2024
**Confidence:** High

## 5. MCP Integration Possibilities

**Claim:** MCP defines stdio transport (client launches server as subprocess, communicates via stdin/stdout JSON-RPC) and Streamable HTTP transport. UE5 plugin could implement MCP client via `CreateProc()` + pipes for stdio, or HTTP for remote servers. MCP JSON-RPC fits naturally with UE5's pipe-based capture.
**URL:** https://modelcontextprotocol.io/specification/latest/basic/transports
**Date:** 2025-06
**Confidence:** High

## 6. Real-World Patterns

**Claim:** Common integration patterns: (a) Build tools (shader compilers, code generators) use `ExecProcess()` for synchronous calls with captured output; (b) Background services use `CreateProc()` with pipes + `FRunnableThread` for async polling; (c) Swarm Agent (lighting build) runs as separate process communicating via named pipes/files.
**URL:** https://forums.unrealengine.com/t/external-processes-t18
**Date:** 2025
**Confidence:** Medium
